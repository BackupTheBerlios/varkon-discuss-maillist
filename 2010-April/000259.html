hello friends,

i'm happy to hear somebody will package varkon for deb based platform.

i have personally given a try and g=came up with a script that checks varkon
out, and builds a package on the fly.

while it's not complete (it especially doesn't cope well with the new way to
load an example project), the varkon installed seems to work (it runs the 6
minute demo).

to allow loading examples, an examples file should be downloadable, probably
with it's name mentioning the varkon version it's good for.
i'd package these examples in a separate deb, and activate the example load on
the fly only when these exist.

my files are attached, in the hope anybody else could use to further expand and
improve.

> at the demos, are as follows:
>
> 1. Ideally, I'd wish to see a collection of demos tested
> against a particular Varkon version. The result could
> be a single archive going to the SourceForge site.
>
> 2. A cautious starting point would be polishing the
> existing demos in relation to an earlier version
> (1.18B?). This would include ensuring all .MBS files are
> included, re-compilation works, and a bit of fiddling
> with license files and other documentation.
>
> 3. Porting forward to 1.19B looks generally easy (though
> I'm not sure about the Fractal Tree). Coming forward to
> 1.19C is more complicated.
>
> 4. If possible, it would be wonderful to also include
> the bicycle optimization example from Gunnar Lidén,
> but porting this one may be harder.
> If you're willing, Ruven, I'd be very happy to correspond
> with you off-list about details of this or whatever plan
> you might prefer.
