Hello Gunnar,

Again this is very helpful, thank you. The strategy of using
MACRO modules to manipulate the active module makes
a lot of sense, and you've provided good clues about how
to do this.

My idea for MACRO code becomes a little strange:
...
FLOAT w;                               ! parameter of interest
INT part_param_index;
INT gm_id, mbs_id;               ! id's of our part,
...
w:=getflt(gm_id, part_parameter_index);
... [interact with user to modify w]
updpp_mbs(mbs_id, part_parameter_index, str(w));

This sequence of taking from the GM but then putting
back through the active module is because the
updpp_mbs() function looks very nice to me, but there
is (at least in 1.19D) no corresponding getpp_mbs(),
and getp_mbs() is refusing to work on a part.

Note that my above example has w as a FLOAT, but I
also want it to work if for example w is in some range
of INT corresponding to choice from get_alt().

How does this getpp_mbs()/updpp_mbs() approach
strike you?

John (MacPhail)
