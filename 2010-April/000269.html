Hello John
 
There is a need for the active module to return results, including errors to
MACROs.
 
The MACRO is a very powerful tool in Varkon and in my tutorial I am trying to
show that with the optimization sample. There results are returned as part
parameter values (Please refer to
http://www.viewsoncad.ch/VarkonTutorial/optim_how.htm).  MACRO modules use these
results to change and execute the active module until the design is optimal.
 
There is room for improvements regarding this “communication” between MACRO and
the active module. Exception handling functionality would for the optimization
application (and other MACRO applications that I have developed) be of minor
use. I would like something else, but for the time being I have no proposals
what it should be like.
 
Geometry (mathematics) was and is still my main interest, but I would be
interested also to contribute (if I can) to a further development of the MACRO
functionality.
   
 
Regards
 
Gunnar
