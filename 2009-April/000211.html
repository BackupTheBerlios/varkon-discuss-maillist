Ok. Thanks Johan, getdat and putdat look like they will work nicely. 

Since I'd like to use a large number of single valued associations I'd like 
to write some macros to help keep the code lean but I'm having some trouble 
getting going.

My questions:

1. Where do I put the macros so they are visible to other code? Currently I 
get "920 E : Unrecognized statement." when I compile a module that refers to 
my macro. I do see the .MBO files in lib.

 2. Is VAR the mechanism to return a value from a macro? I'm just guessing 
from reading some of the MBO files that come with Varkon. 

3. Is there a way to write a macro as a function that returns a single 
value?

I can't test them yet but my macros look something like this:

!store
macro module store_int(STRING key*100;INT varval);
INT k(1);
INT status;
beginmodule
k(1):=varval;
status:=putdat_gm(keyword,1,k);
endmodule

!get
macro module get_int(STRING key*100;VAR INT varval);
INT k(1);
INT status;
beginmodule
status:=getdat_gm(keyword,1,k);
varval=k(1);
endmodule
