> As far as Varkon releases go, I'd like to suggest that we add 1.19B.33
> as the latest release, and use that one for a Debian package. That's the
> one I use all the time. It's stable, everything seems to work, and it
> has some of the latest features. What do you all think?
>
> How about making releases twice a year? I think more frequent releases
> would encourage more users to try the latest versions, and give us more
> feedback.

In general, minimum 6 monthly freeze followed by stable release is
reckoned to greatly promote the quality of the software, for the
knock-on effects of
a) The big releases can follow it (CVS and minor versions are hard to
track) and the Great Unwashed get to use the latest fab version
b) Developers see their work getting into mainstream and are Enthused.
See Michael Michlmayr's thesis on OS s/w quality improvement at
http://cyrius.com/publications/michlmayr-phd.pdf
