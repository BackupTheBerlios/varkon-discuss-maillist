On Tue, 16 Oct 2007, dmm at lemur.com wrote:
>
> If the "settings" information were embedded in the model, then
> each user would pick up previous users' "settings" information,
> and in turn overwrite this with his/her own.  This would be an
> impediment to distributed multiuser access.  (If other systems such
> as Autocad are limited in this way, that would seem simply to be
> an argument in favor of Varkon, not an argument to be like them.)

I must agree! I konw this problem from my practice. I mostly work
with Unigraphics, and there are two classes of user settings. Some
are stored on the workstation environment file (in past versions
plain text, in new XLM - and I must say that text was much easier to
use) and some are stored with model in the part file. This leads to
many problems when passing files between different comapnies.
In Catia V4 there was an idea of 'environment' which was a separete
configuration file and for different task you ave used diferrent env.
files. However in V5 they  have moved to mixed solution similar
to one found in UG. It's even an XML file.
So as a design engineer (not software engineer) I'd like to have the
settings in separate file.
