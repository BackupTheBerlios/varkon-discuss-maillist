I have created a model in hand coded MBS and as it becomes more 
complicated it is becoming difficult to remember the sequence numbers 
for each of the lines and curves.

Is there an elegant way to use names rather than numbers to reference 
entities in MBS?

There are several ways that this can be done but they are all a bit tedious.

eg.

ref mypart;
...
poi_free(#1,...);

mypart := #1;


or

ref n(100);
constant int mypart = 1;
...
poi_free(#1,...);
n(mypart) = #1;


What I really would like to be able to do is:
ref mypart;
...
mypart := poifree(...);


The function add_mbs() does this but it takes strings as parameters and 
requires everything to be quoted.

It would probably be possible to write my program in the style I want 
and then use an external macro processor to convert it to a series of 
quoted add_mbs() statements but that would require writing a parser for 
MBS and would be lot of extra effort.

Is there another solution to this issue?
