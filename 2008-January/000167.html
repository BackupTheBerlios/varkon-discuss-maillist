This is just a short "field report" of sorts.  Varkon svn83 compiles
and works on a 64-bit Kubuntu 7.10 (AMD64) system (running on
hand-assembled AMD Athlon 64 X2 4600+ hardware).
I'm not completely certain, but I think it may have compiled as
a 64-bit application.  An "ldd xvarkon" returns, inter alia,
a link to /lib64/ld-linux-x86-64.so.2, and "readelf -a xvarkon"
shows it to be of class "ELF64".

This was an install onto a working system, not a clean test environment,
so I'm not entirely sure which libraries/packages I had loaded already.
I know that I did have to add, explicitly, packages for:
   libgl1-mesa-dev
   libglu-mesa-dev
   libxpm-dev
   libXent-dev

The only odd thing I had to do to make it work was related to
the fact that this Kubuntu system is offline (none of my working
systems have a connection to the public Internet).  So I had to
add two files by hand:

sources/IG/include/svnversion.h
which contains:
#define SVNVERSION "83"

and
sources/MAN/include/svnversion.inc
which contains
m4_define(_SVNVERSION_,83)

It's nice to have Varkon on my new machine.
My congratulations (and thanks) to the Varkon team.
