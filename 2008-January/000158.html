Some unstructured thoughts:

As you may already have noticed with the pid-files removed etc, a varkon "propject" could no be contained in one folder, which itself can be seen as the job directory with active jobs etc. (for now mbo and job-files but also maybe mbs-files). 
Included in this is folder is the optional lib, doc folders etc needed for the project in question. Note that the lib now contains both mbo- and mbs files for the parts.

This means that it now would be very simple to share a complete project with others. It is only a matter of gathering the directory to a zip or tar file. The receiver can unpack this and then use varkon to open the active job without bothering of changes in the varkon system, pid-files etc.


Now to my question. In the varkon start window you can select to open a module (mbo), which would then be regarded as active module. But what if you only have this as mbs?

I would prefer also to have the option to open an mbs-file as active module. The first thing that I would expect to hapen is the compilation of this to MBO, and then continue as usual.

Maybe (i'm hovever not sure if it is a good idea) the file dialog filter could do this automatically by showing all filenames (without extention) that have MBO and/or MBS extensions. If MBO exist this would be first priority to open. If MBO does not exist, the MBS would open.
(or maybe this will comfuse those who are trying to understand what is going on).

Another even more drastic approach would be to se the active module mbo file as a temporary cache during execution only (at least for the active module). Saving the job would then save only MBS file and open a job would always start by compilation. Starting would be some slower, but during work there would be no difference.



Anyhow, if it was easy ore even standard behavior to open MBS as active module, you could after you have prepared an example for posting to varkon org, strip your folder from all res and mbo -files and. Thereafter create zip-file. The package would then be small and portable.
