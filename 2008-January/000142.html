Hi all,

During my work on an updated Varkon package for Debian a couple of
questions occured.

I am working with an amd64 system.  When trying the examples files
from varkon.org Varkon segfaults with almost any of them.  When
testing the same with a binary built for i386 everything seems to work
fine.  Now, the example projects contain MBO files.  Are MBO files
supposed to be platform independent?  Especially between 32 and 64 bit
platforms?  And since Debian also supports some big endian
architectures - how about those?

Also, when building the package for amd64 I get a lot of compiler
warnings about castings between integers and pointers of different
size.  I would not be surprised if these cause Varkon to crash now and
then.  Are developers testing Varkon on 64 bit architectures?  I am
not really a programmer (let alone C), but I probably could help test
building Varkon on amd64 and find those issues.
