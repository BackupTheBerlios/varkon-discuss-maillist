As part of orienting myself to Varkon, I've looked a bit
at the 1.19D C sources, and have some questions.

1. Am I right that applications typically just use different
start scripts for the program generated by varkon.c,
instead of writing a different main()?

2. If I understand correctly, the Free Software Foundation
argues that it is forbidden for a closed-source program to
statically link to an LGPL library. So it seems strongly
advisable that the default linking of Varkon's libraries
should be made dynamic (.so/.dll). Are there known problems,
or has there been previous discussion about moving to
dynamic linking?

3. I notice that the six libraries {DB, IG, GE, AN, EX, WP}
are all interdependent. In particular, DB depends on IG
for erpush(), and I'm curious as to whether rearrangement
to remove that dependency has been considered.

4. In general, IG is quite confusing for me, even to the
point of being puzzled about the 'G' in the name.
I have a suspicion that IG might better be split into two
or more libraries, but it's also possible that I'm just being
stupid about something I don't understand.
