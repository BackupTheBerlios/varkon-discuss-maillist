John,
 
1. Varkon applications typically never change the C code at all.
A typical application is implemented in MBS. Yes, the start script
is one way to custimize the behaviour of Varkon. Menufiles, init-
modules, exit modules and so on are others.
 
2. Dynamic linking is possible for some libraries, see below.
 
3. Some of the libraries can operate stand alone. erpush() and similar
dependencies
 are available from a VU library (Varkon Utilities). How to use for
example DB stand
 alone (with VU) is shown in the Varkon demo applications.
 
4. Yes IG is not very homogenious. IG is a very old part of Varkon that
made more
sense a long time ago.
 
So, there is demo code available that shows how to use only selected
parts
of Varkon with the VU library. Do you have those demo's ?
