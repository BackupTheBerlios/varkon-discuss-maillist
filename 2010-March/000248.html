Johan Kjellander wrote:
>
> Armin and John and all others..,
>
>  
>
> The Varkon GUI is in the WP library, or at least should be. There are
>
> small parts outside but they can be moved to WP, so basically, moving
>
> Varkon to GTK is porting WP to GTK and replacing WP widgets using
>
> X11/WIN32 with GTK widgets. This may seem easy enough but some
>
> of the WP constructs (dialogs, menus, pop-ups, inputs, buttons...)
>
> are not directly available in GTK. So, porting Varkon to GTK is also
>
> a question of redesigning the Varkon GUI a bit.
>
>  
>
> Given a list of current WP widgets and their functionality, it would be
>
> interesting to hear how GTK would implement similar functionality. If
>
> you want to, I can assemble a list of all WP widgets and short 
> descriptions
>
> of their functionality so that you can have a look.
>
>  
>
> From the GUI point of view, Varkon works as any application with a GUI.
>
> A main loop, polling or waiting for input. On the top level, an input 
> can be
>
> a menu selection or a toolbar button event, each can trigger a specific
>
> Varkon function that will handle it's own interaction with a new loop 
> waiting for
>
> input, triggering new dialogs and so on...
>
>  
>
> On top of this WP also supports two kinds of graphical windows, a 2D 
> window
>
> and a 3D window. With some fixes to the 3D window, the 2D window becomes
>
> obsolete and can be removed from the system. Left is the 3D window 
> which is
>
> based on OpenGL. This window needs to be embedded in a GTK window with
>
> toolbars and command/message line etc.
>
>  
>
> That's about all but we also need to support plotting and printing and
>
> multiple graphical windows simultaneousley. And finally, the new code
>
> needs to be well commented and in a style that is not too different from
>
> the rest of the system.
>
>  
>
> Johan
