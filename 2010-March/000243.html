Hello all,

thanks for digging out the good mail from Carlos Pereira, John.
Johan, I have personal experience using GTK+ but it's a long time
ago - I ported my 3D-measurment app based on stereo images to it
after using raw X11 before.
What I read about GtkGlExt from Carlos contradicts me (my mail to John)
and is releaving, stating it's an alternative to Cairo in rendering
the entire GUI (wrong?).
Appart from what Carlos suggests in our application I see uses for
2 glcontexts: one for the 3D rendering area and one for all the other
widgets that don't need depth buffering etc. I'm not sure how GtkGlExt
windows and those not based on it would interact - his 'area' is a
GtkWidget - the most basic entity in GTK+.
(btw, GL alone can turn on/off buffers like the depth buffer while drawing)

For C I like GTK+. It is object oriented and uses constructs, that
are provided by the Compiler in C++. Otoh, because it's not done
by the compiler, in some cases it can be more flexible.
If there is a way, to make GTK+ use X11 and Win32 (and whatever on Mac OS)
directly OR render all elements via the fastest appropriate method
of OpenGL (e.g. bit-blitting for the 2D stuff in hardware), I'd fully
recommend it. Looking at the GUIs of some games this MUST be fast ;-)

Maybe it's just a missunderstanding of mine: cairo is a rendering engine,
that tries to provide a device independent API to the application 
programmer.
This may make sense for the content display - the view in the
model-view-controller pattern, when speed is no concern.
I can't see where this happens in cario based GTK/GDK - everywhere?
 From what I remember, GDK was a layer to provide device independent 
rendering...
Please - does now GTK build on GDK build on CAIRO?
Why did they not do away with GDK completely, if CAIRO provides
the same service?
Or should we use WP, enhance it's appearance and base that on CAIRO?

All I want is working buttons, a thing I call SwapBox (an intelligent
container widget, where the sheets of a tab can call each other and can
be called by eventhandlers from widgets outside this swapbox), a fast
3D rending window and some basic services like file choosers ;-)
And of course it shall be beautiful and fast...

Best Regards, Armin
