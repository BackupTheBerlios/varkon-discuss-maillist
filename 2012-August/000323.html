Hello Varkon-developers,


I'd like to bring some issues to your attention, which bother me somewhat.

But first I'd like to say, that I am a big fan of Varkon, for it's the only all-purpose parametric FLOSS CAD software I know of, which can be used in 2D and 3D and supports variable views together with dimensioning and hatching.

Unfortunately Varkon has some odd behaviour, which one hast to get used to the hard way.
For example I repeatedly fail in constructing a spherical surface by rotating an full circular arc around an 180° angle. I always have to use a semicircle and rotate it around 360°. Well, that's ok...

Some bigger problem I have is with my current built. I built Varkon on Xubuntu 12.04 and everything went fine so far.
Now, when I create a cylindrical surface by shifting a full circle, I get an strange result, which looks somewhat like this:

             aaa
          aaa   aaa
       aaa         aaa
     aa               aa
    a                   a

The first thingie here is my circular arc I want to shift.

             aaa
          aaa   aaa
       aaa         aac
     aa             b cc
    a              b    c
   a              b      c
  aa             b       cc
   a            b        c
    a          b        c
     aa       b       cc
       aaa   b     ccc
          aac   ccc
             ccc

This is the result I get, with the segment marked as 'c' being the portion of the arc, that has actually been shifted and created the cylindrical surface.

Since I tested all thinkable combinations of entities and parameters without having Varkon give me some hints, I have no real clue what goes wrong.

Some question, that comes to my mind is, whether Varkon is still developed/meintained?

If so, I'd like to recommend two things:

- It would be great for the average user to know which software package delivers the needed dependency libraries for building Varkon. With 'software package' I'm not meaning the specific package that's shipped with the OS-distribution, but the original upstream software and maybe the website that gives further information.

- I noticed, that some (maybe many) of the c-sourcefiles contain commentary in swedish. It would be great for enthusiasts and developers to have some more international language at hand for improving Varkon.
