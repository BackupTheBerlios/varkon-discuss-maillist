Yes, a trimmed surface can be used to represent a surface with a hole.

Representation in database and visualisation are implemented (to appear in next release).

To be able to create a trimmed surface, a low level MBS routine that make it possible to define a trimmed surface is also implemented. It will be used for example by the IGES import of trimmed NURBS surfaces, but it is not suited for interactive use.

To be able to interactively make a hole in a surface, a surface/surface intersection routine is however needed. It will then folow naturaly to add an interactive function for the creation of trimmed surfaces. 

So, in my opinion, the most needed function now to improve Varkon is surface/surface intersection. Surface/plane intersection exist, but could also need some adjustments to make it more reliable.
