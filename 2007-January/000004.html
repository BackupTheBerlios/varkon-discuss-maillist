Regarding NURBS curves

I have not the time right now to investigate exactly, but as i remember it, the implementation status is like this:

Implemented:
- Creation of NURBS curve by CUR_COMPARR()
- Offset of NURBS curves
- Tranformation (tcopy) of NURBS curves


Not implemented:
- Interactive function to create NURBS curves
- Trimming and reversal of NURBS curves
- Use of NURBS curves for surface creation (ruled/rotation/loft).
- Invarcl() does not always work.
- Database function to retrieve NURBS data from an existing curve (compare to getseg() for polynomial curves).

Under develop:
- Use of NURBS (and other curves) as trimming curves for surfaces.


I had thoughts about investigating how to do, and also implement support for the missing items (listed under "not implemented"). It is however not possible to give a time shedule as this is done in "sparetime" and/or when we need this functionallity ourselves. In meantime you can use cur_approx() on the NURBS curve and then use the new curve to create for example lofted surfaces.

The most easy to add i think is database function, interactive function and therafter trimming/reversal by cur_trim(). Some of this is already available internally in the system, but not yet reachable via mbs-functions.
