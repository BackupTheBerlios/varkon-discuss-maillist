Hi Jon,

Great that you're proceeding with the packaging!

1. In general for getting missing files into a Debian install,
package contents can be searched online:
http://www.debian.org/distrib/packages#searchcontents
This quickly solves most build dependencies. However,
there may be differences with Ubuntu, and thought/research
may be needed when different packages are installing
the "same" file. (Small reminder on a related point: when
a file is already installed, use "dpkg -S" to track down which
package installed it.)

2. Running ldd on the generated binaries -- see Johan's post --
is a very useful way to get the runtime dependencies right,
and will often provide useful clues as to what the build
dependencies should be.

3. The Varkon sources include README files which discuss
dependencies, these should make sense against (1) and (2).

4. Comparing your dependencies against the older Debian
package could be an additional check that (1) and (2) are good.

After (1)-(4) above your dependencies should be reliable,
but there may be other things to check that I don't know about
or haven't thought of.

Finally, would you please include unixodbc support? See
sources/README_ODBC. Enabling odbc does mean you will
have a real "diff" from the sources, but I'm guessing it will
not otherwise be trouble. That will increase the certainty
that everything is working. Anybody not wanting odbc and
fussy enough to want a package without it can tweak your
source packages and rebuild.
