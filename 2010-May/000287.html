After going the manuals I must be missing something,
as I still have this question:

Q1. How does my program test whether Varkon is
in explicit or generic mode?

A couple of other questions are more just curiosity:

Q2. Am I guessing correctly that Varkon applications
have usually forced the user into explicit mode?

Q3. Clearing the current MBS file in generic mode seems to
be a little tricky -- the file is open and so can't be overwritten,
we need to avoid trying to delete something already gone,
and it will be a little embarrassing to get an error message
by calling run_mbs() when the MBS file has nothing left.
The following code does seem to work, but is there an "easy"
way to do it instead?

REF p, exempt;
...
  exempt:=global_ref(add_mbs...); ! so job always runs
update:
    run_mbs();
    set_root_gm();
nextp:
    p:=get_next_gm();
    IF p=#0 THEN
        exit();
    ELIF p=exempt THEN
        GOTO nextp;
    ELSE
        del_mbs(p);
        GOTO update;
    ENDIF;
