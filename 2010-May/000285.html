Hi,
 
I agree that it would be rather straight forward and not too cumbersome
to
add things like rounded corners etc. to WP and this would sure make
Varkon
look more modern. Putting more work into WP however is a dead end since
it only runs on X11. Although WP is my baby I'm fully convinced that WP
needs
retirement and that something else must replace WP. GTK is my hope right
now.
 
I'm working through the tutorials on the GTK website and slowly
learning. GTK has
a lot of nice widget features that can be used to build a modern looking
GUI, it
even supports themes. The support for 2D and 3D graphics is on a much
lower
level, very close to the level of WP. This may be a good thing because
porting will
be easier. The big job will be to port all the other WP widgets.

> that can use Cairo as well.
> If this works out of the box... ;-)
> I got a reasonable explanation, why Cairo is so cool in 'geda-user'.
> However, to make Varkon appear more modern, the simplest approach
> would be, to upgrade WP a bit:
> *) define a widget to be a collection of regions
> *) some of these regions are flexible, others fixed size
> *) every widget (region?) has a draw() operation stored in
> a structure-member being a function-pointer
> -> the user of the widget can be agnostic of the draw method(s) just
> leaving them the defaults
> -> if the regions of a given widget are standardized, resource pixmaps
> and ev. size definitions can be exchanged, making the widgets
> themable.
> Example:
> A button would consist of 4 distinct corners, 4 edges and the central
> planar area. Each of them are rendered by copying portions of a
> resource set, so by changing the resource set (and some constants
> defining
> extents) the look of the buttons can be changed.
> Maybe it is like this now... ;-)
> To produce the resource set, any super-heavy uber-method can be used,
> without any impact on runtime performance. It is clear to me, that
> there
> are dynamic requirements, but the eye-candy can be mostly precomputed.
