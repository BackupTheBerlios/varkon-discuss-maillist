Hi Ruven and all of you,

The answer is no. The issue can be generalized to
may other settings as well. Some CAD systems
supply the possibility for the user to create a
"default" job with settings that will always be loaded
when a new job is created. This is definitely something
to add to Varkon 2.0. Start a new job, make all the settings
you like, save the job with a special name (Varkon.JOB or
default.JOB or similar) in a special place ($VARKON_INI)
and this JOB-file will always be loaded prior to a new job.

This leads to another issue, the two Varkon
modes and their different files (*.MBO,*.RES and *.RIT)
In explicit mode (-r on the command line) Varkon acts
as an WYSIWIG system (AutoCAD) and creates *.RIT-files.
No history, no active module, no possibility to re-run.
(You can use parts though)

In generic mode (default mode) a module (*.MBO-file) is
created that you can run at any time to create a new result
(*.RES-file).

How do we communicate this to new users. Suppose
somebody is trying Varkon for the first time. ¨Most probably
this person will start out expecting Varkon to be something
like AutoCAD. It is unlikely that new users will understand
the powerful concept of generic modeling using MBS etc.

So, maybe the default mode for a new job should be explicit.
Alternatively, when a user starts a new job a start menu
with the two alternate modes should be presented ? Finally,
explicit and generic are academic terms, we need better
words.

- "Drafting mode" and "Programming mode" ?

or

- "Edit mode" and "History mode"

or

- "Static mode" and "Variational mode"

or

- "Simple mode" and "Expert mode"

or

- "Documentation mode" and "Modeling mode"

or.....

Also: There is actually a third mode which
is the MBS mode using a text editor.
(Expert++ mode)

Feedback please !

Regards, Johan
